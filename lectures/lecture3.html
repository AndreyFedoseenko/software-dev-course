Classes & objects

In object-oriented programming, a class is a construct that is used to define a distinct type. The class is instantiated into instances of itself â€“ referred to as class instances <...> or simply objects.

State & behaviour

State (or structure) is a data that objects contains and is described using fields

Behavior is a set of rule that shows what can be done with or within the object and is described using methods

How it looks like

						
class Pet {
    // State
    String name;
    int age;
    String breed;

    // Behavior
    void voice() { 
        /* ... */ 
    };
    
    PetEmotion play(PetToy toy) { 
        /* ... */ 
    }
}

Object construction (I)

						
Pet myPet = new Pet();
myPet.name = "Murzik";
myPet.age = 1;
myPet.breed = "Scottish Fold";

Object construction (II)

						
class Pet {
    // State
    String name;
    int age;
    String breed;

    Pet(String name, int age, String breed) {
        this.name = name;
        this.age = age;
        this.breed = breed;
    }

    // Behavior
    /* ... */
}

Pet myPet = new Pet("Murzik", 1, "Scottish Fold");

Encapsulation

  • A language mechanism for restricting access to some of the object's components
  • A language construct that facilitates the bundling of data with the methods (or other functions) operating on that data
						
class Pet {
    public String name;
    protected int age;
    private String breed;

    public PetEmotion scratchTheTummy() { 
        purring();
        return PetEmotion.JOY;
    }

    private void purring() {
         System.out.println("Pur pur pur...")
    }
}

Polymorphism (I): Ad-hoc

  • Polymorphism is the provision of a single interface to entities of different types
  • If a function denotes different and potentially heterogeneous implementations depending on a limited range of individually specified types and combinations, it is called ad hoc polymorphism
						
// Simple examples							
int six = 1 + 2 + 3;
double almostPointSix = 0.1 + 0.2 + 0.3;
String concatinated = "Hello" + ", " + "world!";

// Pet example
class Pet {
    PetEmotion play(Toy toy) { /* ... */ }
    PetEmotion play(Pet otherPet) { /* ... */ }
}

Polymorphism (II): Parametric

  • If the code is written without mention of any specific type and thus can be used transparently with any number of new types, it is called parametric polymorphism. In the object-oriented programming community, this is often called generic programming.
						
List<Integer> listOfInts = new ArrayList<Integer>();
listOfInts.add(1);
listOfInts.remove(0);
listOfInts.size();

List<String> listOfInts = new ArrayList<String>();
listOfInts.add("one");
listOfInts.remove("zero");
listOfInts.size();

Polymorphism (III): Subtyping

  • Subtyping or inclusion polymorphism is a concept wherein a name may denote instances of many different classes as long as they are related by some common superclass
						
class Pet {
    void voice() { System.out.println("Abstract pet voice"); }
}	

class Cat extends Pet {
    @Override
    void voice() { System.out.println("Meow!"); }
}	

class Dog extends Pet {
    @Override
    void voice() { System.out.println("Woof!"); }
}
